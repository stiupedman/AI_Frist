# 三角色工作台功能完善设计

## 目标

在现有家教工作台上按角色精确展示功能，删除管理员、大学生教员、家长学员不需要的按钮和页签，并补齐管理员业务监管、家长查找教员、教员审核状态限制。

采用用户确认的配置驱动方案：一份角色配置决定顶部按钮、默认页签、可见页签、行内操作和数据加载；后端仍执行权限与数据归属校验。

## 角色识别与配置

前端从 Vuex 的 `roles` 获取角色键，按 `admin > tutor > client` 的优先级选择一个工作台配置。每个配置包含：

- `defaultTab`：首次进入的页签。
- `toolbar`：顶部操作按钮。
- `tabs`：可见页签及其数据加载器。
- `actions`：各表格允许显示的行内操作。

管理员不再因为 `*:*:*` 通配权限看到家长和教员业务按钮。权限指令只作为第二层控制，角色配置是界面结构的第一层控制。

大学生教员进入页面时先读取本人资料。资料不存在或 `verifyStatus != '1'` 时，应用“未认证覆盖配置”，只显示资料维护、审核状态和重新提交入口；审核通过后才显示接单相关功能。

## 功能矩阵

### 管理员

- 保留：数据看板、教员审核、投诉处理。
- 新增：业务监管，分别只读查看全平台需求和订单详情。
- 删除：发布需求、申请需求、接受申请、完成授课、评价、收藏、预约、课时记录、个人通知等面向业务参与者的入口。
- 业务监管只提供筛选、刷新和查看，不提供改变订单状态的按钮。

### 大学生教员

- 保留：资料维护、开放需求、智能推荐、我的订单、预约邀请、消息通知、我的投诉。
- 保留操作：申请、撤回申请、接受或拒绝预约、添加课时、完成授课、提交投诉。
- 删除：发布需求、接受教员申请、评价、收藏教员、查找教员。
- 未认证时隐藏全部接单功能，并明确显示未提交、待审核、已驳回或已通过状态及审核意见。

### 家长学员

- 保留：发布需求、我的需求、我的订单、收藏教员、预约邀请、消息通知、我的投诉。
- 新增：找教员，浏览已审核教员，可按科目、学校筛选并查看详情、收藏、预约。
- 保留操作：取消自己的开放需求、接受教员申请、查看课程记录、完成后评价、投诉。
- 删除：无操作价值的“开放需求”、维护教员资料、申请需求、完成授课等教员功能。

## 前端结构与数据流

新增一个纯配置文件保存三种角色的工作台配置，现有 `tutoring/index.vue` 继续复用表格和弹窗，不拆成三个重复页面。

页面初始化流程：

1. 根据 Vuex `roles` 解析当前工作台角色。
2. 教员先加载资料并计算是否已认证。
3. 根据角色和认证状态生成可见按钮、页签和操作。
4. 设置该配置的默认页签。
5. 只加载当前页签需要的数据；切换页签时按配置调用对应加载器。

新增“找教员”和“业务监管”表格，继续使用 Element UI 现有组件和项目请求封装，不引入新依赖。

## 后端接口与权限

新增以下只读能力：

- 家长：分页查询已审核教员，可按科目、学校筛选；返回公开资料和评分，不返回学生证、资格证图片等审核材料。
- 管理员：分页查询全平台需求。
- 管理员：分页查询全平台订单。

现有写接口保持数据归属校验。新增接口分别使用“查找教员”和“业务监管”权限；初始化 SQL 为家长授予查找教员权限，管理员通过超级管理员权限访问监管接口。

管理员监管接口不复用“我的需求/我的订单”查询，避免管理员因用户 ID 过滤只能看到空数据。

## 错误与边界处理

- 角色缺失或不是三种角色时，不渲染家教业务按钮并显示无可用工作台提示。
- 多角色用户按 `admin > tutor > client` 选择唯一界面，避免混合两个角色的操作。
- 教员资料加载失败时保持锁定状态，不提前开放接单功能。
- 当前页签因角色或审核状态变化而不可用时，自动回到该角色默认页签。
- 前端隐藏不替代后端校验；越权请求仍返回 403，跨用户写操作仍由服务层拒绝。

## 验证标准

- 管理员只看到数据看板、业务监管、教员审核、投诉处理；监管表格能看到全平台数据且没有业务写按钮。
- 已认证教员能完成申请、撤回、预约处理、课时记录和完成授课；未认证教员只看到资料与审核状态。
- 家长看不到开放需求，能从“找教员”查看已认证教员并收藏或预约，仍可管理自己的需求、订单、评价和投诉。
- 三种角色切换页签不会请求不属于该角色的接口。
- 后端构建和前端生产构建通过；保留一个可直接运行的角色配置断言检查，覆盖三角色可见项和教员锁定状态。

## 不在本次范围

- 不拆分三个独立工作台页面。
- 不增加聊天、支付、排课日历或新的业务角色。
- 不重构与家教工作台无关的若依系统管理页面。
